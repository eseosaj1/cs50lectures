ATO Background Information

ATO process is a comprehensive assessment and/or evaluation of an information system policies, technical and non-technical security components, documentation, supplemental safeguards, policies, and vulnerabilities. The ATO process establishes the extent to which a particular design and control implementation, meet a set of specified security requirements defined by the organization, government guidelines, and federal mandates into a formal authorization package. This authorization package is reviewed by the Authorizing Official (AO) and a formal declaration of an information system accreditation is either granted as an Authorization to Operate (ATO) or ATO with conditions or outright denial of authorization to operate.

Given the ATO the information system is to operate in a particular security mode using a prescribed set of safeguards and function at an acceptable level of risk to the agency. Each information system is placed into the information security continuous monitoring program which maintains the ongoing awareness of information security vulnerabilities, and threats to an information system.

ATO Steps/Phases

  1. Categorization:

    Categorize the information systems, i.e., determine the criticality of the information system based on potential adverse impact to the business, to either:
    • High risk category
    • Moderate risk category
    • or Low risk category

  2. Security control selection:

    Select baseline security controls, using NIST 800-53 controls as guide.

  3. Implement security control:

    Implement these security controls, i.e., implement security controls within the business network environment:-
    • - system owner would implement and document selected control
    • - system owner would do a self assessment of implemented controls and remediate any findings
    • - when satisfy, system will be turn over to technical team for security/technical assessment

  4. Security Assessment:

    Assess the security controls to determine their effectiveness.

  5. Authorization:

    Authorize the system to operate in the enterprise network environment.

  6. Continuous Monitoring:

    monitor authorized system on a continous basis for control effectiveness
